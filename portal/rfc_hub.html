ℹ️ How Branch Protection Works
1
Developer pushes to staging branch only — never directly to master.
2
build-check-merge.yml auto-triggers: NuGet restore + MSBuild compile on GitHub cloud runner (windows-latest).
3
✅ Build passes → auto-merges staging → master. ❌ Build fails → master is never touched. Developer fixes and re-pushes staging.
4
deploy-iis.yml triggers on master push — runs entirely on .36 self-hosted runner: checkout → NuGet → MSBuild → copy DLLs → recycle V2RfcTestPool → health check.

15-Point Review Checklist — Applied to Every Generated Controller
CRITICAL — Deploy blocked if any fail
  1. STRUCTURE vs TABLE: export param kind matches code (GetStructure/GetTable)
  2. TABLE INPUT: Append()+SetValue() — no CreateStructure()
  3. SAP connector pattern: rfcConfigparameters→GetDestination→CreateFunction→Invoke
  4. GetString() field names match SAP spec EXACTLY (case, spelling)
  5. Invoke() before any Get*() call
HIGH — Score penalty
  1. Import param names match RFC spec
  2. Required params validated before SAP call
  3. [Route("api/{ExactRfcName}")] matches function
  4. Correct namespace + using BaseController
  5. Try/catch: RfcAbapException, CommunicationException, Exception
MEDIUM
  1. Response has Status + Message
  2. Table RFC has RecordCount
  3. No dynamic/reflection for SAP reading
  4. All GetString keys are UPPERCASE
  5. Request model matches import param names
Min score: 9/10  ·  Up to 3 iterations of review+fix  ·  Full rewrite if score < 8
