POST /ZPO_DD_UPD_RFC/Post Update PO Delivery Date ▾
Parameters
cURL
Response Schema
Try It

Updates the delivery date on a Purchase Order in SAP ECC 6.0. Writes directly to SAP — confirm PO number before calling.

NameSAP TypeRequiredDescription
PO_NO EBELN required Purchase Order number — 10 digits
DELV_DATE CHAR10 required New delivery date — format YYYYMMDD or DD.MM.YYYY

EXPORT FIELDS

NameSAP TypeDescription
MSG_TYPECHAR1S = Success, E = Error, W = Warning
MESSAGECHAR100Human-readable SAP response message
curl -s -X POST "/ZPO_DD_UPD_RFC/Post" \ -H "Content-Type: application/json" \ -d '{ "PO_NO": "4500001234", "DELV_DATE": "20260401" }' # PowerShell (from .36 server) $body = '{"PO_NO":"4500001234","DELV_DATE":"20260401"}' Invoke-WebRequest "http://192.168.151.36:9292/api/ZPO_DD_UPD_RFC/Post" ` -Method POST -Body $body -ContentType "application/json" -UseBasicParsing

Success — delivery date updated in SAP

{ "Status": true, "MSG_TYPE": "S", "MESSAGE": "Delivery date updated for PO 4500001234" }

SAP Business Error (PO not found)

{ "Status": false, "MSG_TYPE": "E", "MESSAGE": "This line is not contained in the table." }
POST /ZPO_MODIFICATION/Execute PO Modification History ▾
Parameters
cURL
Response Schema
Try It

Returns PO modification history within a date range. All parameters optional — use date range to filter by change date.

IMPORT PARAMETERS

NameSAP TypeRequiredDescription
IM_PO_NOEBELNoptionalFilter by specific PO number
IM_PO_DEL_DATEEINDToptionalFilter by delivery date on PO item
IM_DEL_CHG_DATE_LOWERDAToptionalDate range — from (YYYYMMDD)
IM_DEL_CHG_DATE_HIGHERDAToptionalDate range — to (YYYYMMDD)

ET_PO_OUTPUT TABLE (ZPO_MODIFICATION_TT)

EBELN CHAR10 · PO Number
ORIGNAL_DEL_DATE CHAR50
CHNG_NO CHAR10 · Change No.
CURRENT_DEL_DATE DATS8
DEL_EXT_DATE CHAR10
DELAYED_BY CHAR10 · Days
REASON CHAR100
curl -s -X POST "/ZPO_MODIFICATION/Execute" \ -H "Content-Type: application/json" \ -d '{ "IM_PO_NO": "", "IM_DEL_CHG_DATE_LOW": "20260101", "IM_DEL_CHG_DATE_HIGH":"20260319" }'
{ "Status": true, "Data": { "ET_PO_OUTPUT": [ { "EBELN": "4500001234", "ORIGNAL_DEL_DATE": "20260101", "CHNG_NO": "001", "CURRENT_DEL_DATE": "20260315", "DEL_EXT_DATE": "20260315", "DELAYED_BY": "73", "REASON": "Production delay" } ] } }
POST /ZADVANCE_PAYMENT_RFC/Post Advance Payment FI Documents ▾
Parameters
cURL
Response Schema
Try It

Fetches advance payment FI documents for a company code within a posting date range. Returns IT_FINAL table of type ZADVANCE_TT — 16 fields per row.

IMPORT PARAMETERS

NameSAP TypeRequiredDescription
I_COMPANY_CODE BUKRS required Company Code (e.g. 1100)
I_POSTING_DATE_LOW BUDAT required Posting Date From — format: YYYYMMDD
I_POSTING_DATE_HIGH BUDAT required Posting Date To — format: YYYYMMDD

IT_FINAL OUTPUT FIELDS (ZADVANCE_ST structure — 16 fields)

DOCUMENT_TYPE BLART · CHAR 2
COMPANY_CODE BUKRS · CHAR 4
DOCUMENT_NUMBER H_BELNR · CHAR 12
FISCAL_YEAR GJAHR · NUMC 4
LINE_ITEM BUZEI · NUMC 3
POSTING_KEY BSCHL · CHAR 2
ACCOUNT_TYPE KOART · CHAR 1
SPECIAL_G_L_IND UMSKZ · CHAR 1
TRANSACT_TYPE UMSKS · CHAR 1
DEBIT_CREDIT SHKZG · CHAR 1
AMOUNT_IN_LC DMBTR · CURR 23,2
AMOUNT WRBTR · CURR 23,2
TEXT SGTXT · CHAR 50
VENDOR LIFNR · CHAR 10
PAYMENT_AMT NEBTR · CURR 23,2
POSTING_DATE BUDAT · DATS 8
curl -s -X POST "/ZADVANCE_PAYMENT_RFC/Post" \ -H "Content-Type: application/json" \ -d '{ "I_COMPANY_CODE": "1100", "I_POSTING_DATE_LOW": "20260101", "I_POSTING_DATE_HIGH": "20260319" }' | python3 -m json.tool # PowerShell $body = '{ "I_COMPANY_CODE": "1100", "I_POSTING_DATE_LOW": "20260101", "I_POSTING_DATE_HIGH": "20260319" }' Invoke-WebRequest "http://192.168.151.36:9292/api/ZADVANCE_PAYMENT_RFC/Post" ` -Method POST -Body $body -ContentType "application/json" -UseBasicParsing

Success — returns array of advance payment document lines
